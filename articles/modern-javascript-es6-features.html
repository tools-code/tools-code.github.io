Modern JavaScript: ES6+ Features You Should Know

January 10, 2025 • JavaScript

JavaScript has evolved significantly over the years, with ES6 (ECMAScript 2015) introducing many new features that have transformed how we write code. In this article, we'll explore the most important ES6+ features that every JavaScript developer should know.

Let and Const Declarations

ES6 introduced two new ways to declare variables: let and const.

// Before ES6
var name = 'John';
var age = 30;

// ES6+
let name = 'John'; // Block-scoped
const age = 30; // Block-scoped and immutable

The key differences:

  • var is function-scoped, while let and const are block-scoped.
  • const declarations cannot be reassigned, but objects and arrays declared with const can still be modified.

Arrow Functions

Arrow functions provide a more concise syntax for writing function expressions.

// Traditional function
function add(a, b) {
    return a + b;
}

// Arrow function
const add = (a, b) => a + b;

// Arrow function with multiple statements
const add = (a, b) => {
    const sum = a + b;
    return sum;
};

// Arrow function with a single parameter
const square = x => x * x;

Arrow functions also have lexical this binding, meaning they inherit this from the enclosing scope.

Template Literals

Template literals allow for easier string interpolation and multi-line strings.

// Before ES6
const name = 'John';
const message = 'Hello, ' + name + '!';

// ES6+
const name = 'John';
const message = `Hello, ${name}!`;

// Multi-line strings
const multiLine = `
    This is a
    multi-line string
`;

Destructuring Assignment

Destructuring allows you to extract values from objects and arrays into variables.

Object Destructuring

const person = {
    name: 'John',
    age: 30,
    address: {
        city: 'New York',
        country: 'USA'
    }
};

// ES6+
const { name, age } = person;
console.log(name); // 'John'
console.log(age); // 30

// With default values
const { name, age, occupation = 'Developer' } = person;

// Nested destructuring
const { address: { city } } = person;
console.log(city); // 'New York'

Array Destructuring

const numbers = [1, 2, 3, 4, 5];

// ES6+
const [first, second] = numbers;
console.log(first); // 1
console.log(second); // 2

// With rest operator
const [first, ...rest] = numbers;
console.log(rest); // [2, 3, 4, 5]

// With default values
const [first, second, third = 3] = [1, 2];

Spread Operator

The spread operator (...) allows you to expand arrays and objects.

Arrays

const numbers1 = [1, 2, 3];
const numbers2 = [4, 5, 6];

// Concatenating arrays
const combined = [...numbers1, ...numbers2]; // [1, 2, 3, 4, 5, 6]

// Creating a copy
const copy = [...numbers1];

// Passing arguments
const max = Math.max(...numbers1); // 3

Objects

const person = {
    name: 'John',
    age: 30
};

// Creating a copy
const copy = { ...person };

// Merging objects
const updated = {
    ...person,
    age: 31,
    occupation: 'Developer'
};

Rest Parameters

Rest parameters allow functions to accept an indefinite number of arguments as an array.

function sum(...numbers) {
    return numbers.reduce((total, num) => total + num, 0);
}

console.log(sum(1, 2, 3)); // 6
console.log(sum(1, 2, 3, 4, 5)); // 15

Default Parameters

Default parameters allow you to specify default values for function parameters.

function greet(name, greeting = 'Hello') {
    return `${greeting}, ${name}!`;
}

console.log(greet('John')); // 'Hello, John!'
console.log(greet('John', 'Hi')); // 'Hi, John!'

Modules

ES6 introduced a standard module system for JavaScript.

Exporting

// Exporting individual items
export const PI = 3.14;
export function add(a, b) {
    return a + b;
}

// Exporting a default
export default function multiply(a, b) {
    return a * b;
}

Importing

// Importing individual items
import { PI, add } from './math.js';

// Importing a default
import multiply from './math.js';

// Importing everything
import * as math from './math.js';
console.log(math.PI);
console.log(math.add(1, 2));

Classes

ES6 introduced class syntax, which is syntactical sugar over JavaScript's prototypal inheritance.

class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
    
    greet() {
        return `Hello, my name is ${this.name}`;
    }
    
    static create(name, age) {
        return new Person(name, age);
    }
}

class Employee extends Person {
    constructor(name, age, jobTitle) {
        super(name, age);
        this.jobTitle = jobTitle;
    }
    
    greet() {
        return `${super.greet()} and I'm a ${this.jobTitle}`;
    }
}

const person = new Person('John', 30);
const employee = new Employee('Jane', 25, 'Developer');

Promises

Promises provide a way to handle asynchronous operations.

// Creating a promise
const fetchData = () => {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const data = { name: 'John', age: 30 };
            resolve(data);
            // or reject(new Error('Failed to fetch data'));
        }, 1000);
    });
};

// Using a promise
fetchData()
    .then(data => {
        console.log(data);
        return data.age;
    })
    .then(age => {
        console.log(`Age: ${age}`);
    })
    .catch(error => {
        console.error(error);
    })
    .finally(() => {
        console.log('Operation completed');
    });

Async/Await

Async/await is syntactical sugar over promises, making asynchronous code look more synchronous.

async function getData() {
    try {
        const data = await fetchData();
        console.log(data);
        const age = data.age;
        console.log(`Age: ${age}`);
        return data;
    } catch (error) {
        console.error(error);
    } finally {
        console.log('Operation completed');
    }
}

getData();

ES7+ Features

Array Includes

const numbers = [1, 2, 3, 4, 5];
console.log(numbers.includes(3)); // true
console.log(numbers.includes(6)); // false

Exponentiation Operator

console.log(2 ** 3); // 8
console.log(3 ** 2); // 9

Object Entries and Values

const person = {
    name: 'John',
    age: 30
};

console.log(Object.keys(person)); // ['name', 'age']
console.log(Object.values(person)); // ['John', 30]
console.log(Object.entries(person)); // [['name', 'John'], ['age', 30]]

Nullish Coalescing Operator

// Returns the right-hand side if the left-hand side is null or undefined
const name = null ?? 'Default';
console.log(name); // 'Default'

// Unlike ||, it doesn't consider falsy values like 0, '', or false
const count = 0 ?? 10;
console.log(count); // 0

Optional Chaining

const person = {
    name: 'John',
    address: {
        city: 'New York'
    }
};

// Safely access nested properties
console.log(person?.address?.city); // 'New York'
console.log(person?.contact?.phone); // undefined

// With function calls
console.log(person?.greet?.()); // undefined

Conclusion

ES6+ has brought many powerful features to JavaScript that have made the language more expressive, concise, and maintainable. By mastering these features, you'll be able to write cleaner, more efficient code and take advantage of the latest JavaScript capabilities.

Remember that while these features are widely supported in modern browsers and Node.js environments, you may need to use a transpiler like Babel for older environments.
